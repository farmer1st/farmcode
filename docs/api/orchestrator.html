Orchestrator Service (0.1.0)

Download OpenAPI specification:

The Orchestrator Service manages SpecKit workflow execution. It owns workflow definitions (specify, plan, tasks, implement) and manages state transitions. All agent invocations go through the Agent Hub Service.

workflows

Create Workflow

Create and start a new workflow.

Args: request: CreateWorkflowRequest with workflow_type and feature_description db: Database session

Returns: WorkflowResponse with created workflow

Raises: HTTPException: On validation error

Request Body schema: application/json
required
workflow_type
required
string (Workflow Type)

Type of workflow: specify, plan, tasks, implement

feature_description
required
string (Feature Description) >= 10 characters

Description of the feature

Context (object) or Context (null) (Context)

Additional context for the workflow

Advance Workflow

Advance workflow to next state.

Args: workflow_id: Workflow UUID request: AdvanceWorkflowRequest with trigger and optional phase_result db: Database session

Returns: WorkflowResponse with updated workflow

Raises: HTTPException: On error

path Parameters
workflow_id
required
string (Workflow Id)
Request Body schema: application/json
required
trigger
required
string (Trigger)

Trigger: agent_complete, human_approved, human_rejected

Phase Result (object) or Phase Result (null) (Phase Result)
